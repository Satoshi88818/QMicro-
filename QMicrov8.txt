qmicro/src/sync.rs — UPDATED [IMP-v8-1]

Fix: The v7 implementation wrapped spin::Mutex, which is not IRQ-safe on single-core Cortex-M. If a low-priority thread holds the spinlock and a hardware interrupt fires that tries to acquire the same lock, the system deadlocks permanently.

The fix: use cortex_m::interrupt::free (which sets PRIMASK, globally masking all interrupts) for the duration of the critical section. The API now accepts a closure instead of returning a guard, preventing accidental guard leakage across function boundaries.

//! True IRQ-safe mutex for bare-metal single-core Cortex-M. //! //! # [IMP-v8-1] Fix: Real interrupt safety //! //! Uses `cortex_m::interrupt::free` (sets PRIMASK) for the duration of the //! critical section — no spinning, no deadlock from ISR re-entry. //! //! ## API change //! Lock now accepts a **closure** (idiomatic bare-metal Rust): //! //! ```rust //! // Before (v7, unsafe guard): //! let mut guard = MUTEX.lock(); //! guard.field = 42; //! //! // After (v7+IMP-v8-1, closure): //! MUTEX.lock(|inner| inner.field = 42); //! ``` use core::cell::UnsafeCell; /// A mutex that disables all interrupts (sets PRIMASK) while the inner value /// is accessed. Safe for use on single-core Cortex-M from both thread and /// interrupt contexts. pub struct IrqSafeMutex<T>(UnsafeCell<T>); // Safety: on single-core Cortex-M, PRIMASK exclusion is sufficient for Sync. unsafe impl<T: Send> Sync for IrqSafeMutex<T> {} impl<T> IrqSafeMutex<T> { /// Create a new mutex. `const fn` so it can initialise statics. pub const fn new(val: T) -> Self { Self(UnsafeCell::new(val)) } /// Execute `f` with exclusive access to the inner value. /// /// Interrupts are globally disabled (PRIMASK set) for the duration of `f` /// and restored atomically on exit, even if `f` panics. /// /// # Reentrancy /// Do **not** call `lock` recursively from within `f`. #[inline] pub fn lock<R, F: FnOnce(&mut T) -> R>(&self, f: F) -> R { cortex_m::interrupt::free(|_cs| { // Safety: interrupts are disabled; exclusive access is guaranteed. f(unsafe { &mut *self.0.get() }) }) } } 

qmicro/src/capability.rs — UPDATED [IMP-v8-7]

Fix: The v7 implementation wrapped a heapless::spsc::Queue in an IrqSafeMutex, defeating the purpose of an SPSC queue and creating re-entrancy deadlock risk in OwnedCap::drop.

The fix: one lock-free SPSC queue per domain. Each domain is the sole producer for its own queue slot; the idle-pass consumer never runs concurrently with that producer. OwnedCap::drop is now zero-lock on the hot path.

qmicro/src/scheduler.rs — UPDATED [IMP-v8-5]

} struct P4Ring { buf: [PulseDescriptor; RING_SIZE], head: AtomicUsize, tail: AtomicUsize, } impl P4Ring { const fn new() -> Self { Self { buf: [PulseDescriptor::ZERO; RING_SIZE], head: AtomicUsize::new(0), tail: AtomicUsize::new(0), } } fn push(&mut self, desc: PulseDescriptor) -> KResult<()> { let tail = self.tail.load(Ordering::Relaxed);
